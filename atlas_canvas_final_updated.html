ğŸ“‹ Product Overview

Atlas is an enterprise-grade Retrieval-Augmented Generation (RAG) system designed to solve the fundamental challenge of making sense of massive research document collections. Unlike traditional search engines that simply match keywords, Atlas understands meaning, extracts knowledge, and visualizes relationships across 10,000+ research papers spanning AI/ML, Finance, Blockchain, Transportation, Healthcare, and Sports.

ğŸ’¡ Value Proposition

The Problem

Research teams and analysts face critical information management challenges:

  1. Information Overload: Thousands of PDFs scattered across file systems with no unified, intelligent search capability
  2. Lost Connections: Related concepts, similar methodologies, and cross-domain insights remain undiscovered because they're buried in different documents
  3. Citation Burden: Manually tracking sources, verifying claims, and maintaining reference accuracy is extremely time-intensive
  4. Knowledge Silos: Research exists in isolated documents rather than as an interconnected knowledge network

The Atlas Solution

Atlas transforms document chaos into structured, queryable knowledge:

  • Semantic Search: Natural language queries return contextually relevant passages, understanding meaning beyond keywords
  • Entity Extraction & Linking: Automatically identifies people, organizations, technologies, locations, and concepts using spaCy's en_core_web_lg model
  • Knowledge Graph Visualization: Interactive Cytoscape-based graphs showing entity relationships, with color-coded nodes and double-click interactions
  • Automatic Citation: Every response includes precise [DOC_ID:CHUNK_ID] references for instant verification
  • Multi-Hop Reasoning: Answer complex questions requiring synthesis from multiple papers across different domains using graph traversal

Measurable Impact

Research Efficiency

  • 10x faster literature exploration vs. manual reading
  • <150ms query response times on 200K chunks
  • Automatic source tracking eliminates citation errors

Knowledge Discovery

  • 50K+ entities extracted and linked automatically
  • Cross-domain connections surface novel insights
  • Visual graphs with legend accelerate understanding

ğŸ¯ Key Differentiators

What Sets Atlas Apart

1. Hybrid Architecture: The Best of Both Worlds

Most RAG systems use either vector search OR knowledge graphs. Atlas uses BOTH simultaneously:

  • PostgreSQL + pgvector: Handles semantic similarity with IVFFlat indexing for sub-150ms queries on 200K chunks
  • Neo4j: Manages entity relationships and graph traversal for explainable connections with 50K+ unique entities
  • Integrated Reasoning: The search_rag_plus_graph endpoint combines both systems seamlessly

2. Optimized Data Pipeline

Built for scale using Google Colab GPU acceleration:

  • Cloud-based embedding generation: Free GPU on Colab for batch processing (64 embeddings at a time)
  • Fast local storage: Samsung T7 external SSD for high-throughput PDF access
  • Parquet format: Efficient columnar storage for embeddings with bulk ingestion scripts
  • Smart chunking: 300-token chunks with 50-token overlap preserve context

3. Production-Scale Engineering

Built for real-world deployment, not just demos:

  • Handles 10,000+ documents with sub-150ms latency
  • Robust duplicate detection via SHA-256 hashing
  • Vector indexing (IVFFlat with 100 lists) for scalable similarity search
  • Docker Compose for one-command deployment
  • ColBERT + BM25 hybrid search options for flexibility

4. Rich Interactive Visualization

Graph exploration with professional UI/UX:

  • Color-coded entities: Documents (blue), Chunks (gray), Persons (green), Organizations (orange), Technologies (purple), Locations (red), Concepts (yellow)
  • Interactive legend: Real-time visual guide to node types
  • Double-click actions: Documents open PDFs, chunks show previews
  • Chat history sidebar: Track past queries and results

Design Rationale

Why Hybrid PostgreSQL + Neo4j?

PostgreSQL + pgvector

Use Case: Semantic similarity search

  • Optimized ANN retrieval (<150ms)
  • ACID guarantees for data integrity
  • Excellent Python integration
  • Hybrid queries (vector + structured)

Neo4j

Use Case: Relationship reasoning

  • Native graph traversal (Cypher)
  • Visualize entity networks
  • Multi-hop queries
  • Explain connections

Why Google Colab for Embeddings?

  • Free GPU access: Generate 200K+ embeddings without hardware investment
  • Batch processing: Process 64 chunks at once for efficiency
  • Parquet export: Efficient columnar format for bulk local ingestion
  • Separation of concerns: Generate embeddings once, reuse locally

Why Local SSD Storage?

  • High throughput: Samsung T7 delivers ~1000 MB/s read speeds
  • Cost-effective: No cloud storage fees for 10,000+ PDFs
  • Privacy: All data remains on-premises
  • Reliability: Direct file access without network dependency

Key Implementation Details

Chunking Strategy (300 tokens, 50 overlap)

def chunk_text(text: str, chunk_size=300, overlap=50):
    """
    Splits text into overlapping chunks to preserve 
    context at boundaries.
    
    Args:
        text: Input text to chunk
        chunk_size: 300 tokens per chunk
        overlap: 50 overlapping tokens
    
    Returns:
        List of text chunks
    """
    tokenizer = AutoTokenizer.from_pretrained(
        "sentence-transformers/all-MiniLM-L6-v2"
    )
    tokens = tokenizer.encode(text)
    
    chunks = []
    for i in range(0, len(tokens), chunk_size - overlap):
        chunk = tokenizer.decode(
            tokens[i:i + chunk_size]
        )
        chunks.append(chunk)
    
    return chunks

Embedding Generation (Batch 64)

from sentence_transformers import SentenceTransformer

model = SentenceTransformer(
    "sentence-transformers/all-MiniLM-L6-v2"
)

# Batch processing for efficiency (64 at once)
BATCH_SIZE = 64
embeddings = []

for i in range(0, len(chunks), BATCH_SIZE):
    batch = chunks[i:i + BATCH_SIZE]
    batch_embeddings = model.encode(batch)
    embeddings.extend(batch_embeddings)

# embeddings shape: (N, 384)

Vector Search with pgvector

async def semantic_search(query: str, k: int = 10):
    """Perform semantic similarity search"""
    # Generate query embedding
    query_embedding = model.encode(query).tolist()
    
    # Vector similarity search with L2 distance
    results = await db.execute(
        """
        SELECT c.id, c.text,
               c.embedding <-> %s::vector AS distance
        FROM chunk c
        ORDER BY c.embedding <-> %s::vector
        LIMIT %s
        """,
        (query_embedding, query_embedding, k)
    )
    
    return results  # <150ms on 200K chunks

Entity Extraction with spaCy

import spacy

# Load large English model
nlp = spacy.load("en_core_web_lg")

def extract_entities(text: str):
    """Extract named entities using spaCy"""
    doc = nlp(text)
    
    entities = []
    for ent in doc.ents:
        entities.append({
            "name": ent.text,
            "type": ent.label_,  # PERSON, ORG, GPE, etc.
            "start": ent.start_char,
            "end": ent.end_char
        })
    
    return entities  # ~50K unique across corpus
